# 基于情绪的智能音乐播放器 - 期末大作业说明书

---

## 第一章 项目概述

### 1.1 项目背景

随着数字音乐消费的普及，音乐播放器已经成为人们日常生活中不可或缺的工具。传统的音乐播放器大多基于固定的分类方式（如歌手、专辑、流派等）进行音乐管理，缺乏对用户情感状态的智能感知与响应。

情绪音乐推荐系统是一种新兴的音乐服务模式，它能够根据用户当前的情绪状态，智能推荐符合用户情感需求的音乐。这种方式不仅提升了音乐发现的效率，还能在一定程度上帮助用户调节情绪，获得更好的聆听体验。

### 1.2 项目目标

本项目旨在开发一款基于情绪的智能音乐播放器，主要实现以下目标：

1. **情绪识别与分类**：实现多种情绪状态的识别，包括快乐、悲伤、平静、兴奋等预定义情绪，以及用户自定义情绪功能。

2. **智能音乐推荐**：基于用户选择的情绪，智能推荐相应的音乐，提供个性化的音乐服务体验。

3. **完整的播放功能**：实现音乐播放、暂停、进度控制、上一首/下一首等完整的播放器功能。

4. **跨平台支持**：基于Electron框架实现Windows桌面应用，同时预留Capacitor移动端框架支持。

5. **用户界面设计**：采用现代化UI设计，支持明暗主题切换，适配桌面端和移动端设备。

### 1.3 项目意义

1. **提升用户体验**：通过情绪推荐，用户无需手动筛选音乐，系统能够自动匹配符合当前情绪的音乐。

2. **情绪调节功能**：音乐对情绪有显著的调节作用，情绪化推荐能够帮助用户更好地管理情绪状态。

3. **技术实践**：本项目综合运用了React框架、TypeScript类型系统、TailwindCSS样式设计、Zustand状态管理、Electron桌面开发等多项前端技术，是一次全面的技术实践。

## 第三章 功能实现

### 3.1 情绪选择系统

#### 3.1.1 预定义情绪

系统内置了10种预定义情绪，每种情绪对应特定的音乐标签：

| 情绪 | 表情 | 对应标签 |
|------|------|----------|
| Happy（快乐） | 😊 | happy, upbeat, cheerful, joyful |
| Sad（悲伤） | 😢 | sad, melancholic, emotional, ballad |
| Calm（平静） | 😌 | calm, relaxing, peaceful, ambient |
| Excited（兴奋） | 🔥 | excited, energetic, party, dance |
| Energetic（活力） | 💪 | energetic, workout, power, intense |
| Romantic（浪漫） | 💖 | romantic, love, intimate, slow |
| Melancholic（忧郁） | 🌙 | melancholic, reflective, nostalgic |
| Relaxed（放松） | 🌟 | relaxed, chill, easy, laid-back |
| Focused（专注） | 🎯 | focused, study, instrumental, concentration |
| Random（随便听听） | 🎶 | 未分类或标签为none的歌曲 |

#### 3.1.2 自定义情绪

用户可以通过以下步骤创建自定义情绪：

1. 点击情绪选择器中的 "+" 按钮
2. 在弹出的模态框中输入情绪名称（最多10个字符）
3. 选择一个表情符号（提供10种可选）
4. 选择背景颜色（10种颜色可选）
5. 点击"确认创建"按钮

自定义情绪创建后，系统会根据用户输入的情绪名称作为标签进行歌曲匹配。例如，创建名为"摇滚"的情绪，系统会匹配所有标签中包含"摇滚"的歌曲。

### 3.2 音乐推荐算法

#### 3.2.1 预定义情绪推荐

对于预定义情绪，系统采用以下推荐策略：

1. **精确匹配**：优先选择 `mood` 字段与当前情绪精确匹配的歌曲
2. **标签评分**：根据歌曲标签与情绪标签的重合度计算匹配分数
3. **播放次数排序**：同分数歌曲按播放次数降序排列

#### 3.2.2 自定义情绪推荐

对于自定义情绪，系统采用以下推荐策略：

1. **名称匹配**：使用自定义情绪的名称作为关键词
2. **模糊匹配**：匹配歌曲标签中包含情绪名称的歌曲
3. **包含未分类歌曲**：同时包含未分类（mood='none'）的歌曲
4. **匹配度排序**：按标签匹配程度和播放次数综合排序

### 3.3 音频播放控制

#### 3.3.1 全局音频对象

为了解决多组件间音频对象不一致的问题，系统采用全局单例模式管理音频对象：

- 使用模块级变量 `globalAudioRef` 存储全局音频实例
- 提供 `getGlobalAudio()` 函数获取全局音频对象
- 所有组件共享同一个音频实例，确保播放状态同步

#### 3.3.2 进度条控制

进度条控制是本项目的技术难点之一，主要实现以下功能：

1. **实时进度显示**：通过 `timeupdate` 事件实时更新播放进度
2. **点击跳转**：用户点击进度条任意位置，跳转到对应时间
3. **拖动控制**：用户拖动滑块时可自由调整播放位置
4. **状态管理**：使用 `isSeeking` 标志避免在拖动时与播放时间更新冲突

关键代码实现：

```typescript
// 全局 seek 状态
let isSeeking = false;

// seek 函数
export function seekAudio(time: number) {
  const audio = getGlobalAudio();
  if (!audio) return;

  isSeeking = true;
  audio.currentTime = time;

  // 500ms 后重置状态（防止 seeked 事件未触发）
  setTimeout(() => {
    isSeeking = false;
  }, 500);
}

// 时间更新处理
const handleTimeUpdate = () => {
  if (isSeeking) return;  // 拖动时不更新
  // 更新当前播放时间
};
```

### 3.4 主题切换系统

#### 3.4.1 明暗主题

系统支持白天模式和黑夜模式两种主题：

**白天模式特点**：
- 主背景：明亮的白色 (`bg-slate-50`)
- 标题文字：深色 (`text-slate-900`)
- 卡片背景：白色半透明 (`bg-white/90`)
- 进度条：浅灰色轨道 (`bg-slate-300`)

**黑夜模式特点**：
- 主背景：深灰蓝色 (`bg-slate-900`)
- 标题文字：浅色 (`text-white`)
- 卡片背景：深色半透明 (`bg-slate-800/80`)
- 进度条：深色轨道 (`bg-slate-700`)

#### 3.4.2 主题切换按钮

主题切换按钮采用状态显示设计：

- 显示当前主题图标（☀️ 白天 / 🌙 黑夜）
- 显示当前主题文字（"白天模式" / "黑夜模式"）
- 鼠标悬停显示提示："点击切换到白天/黑夜模式"
- 点击切换主题

### 3.5 文件导入功能

#### 3.5.1 支持的音频格式

系统支持导入以下常见音频格式：

- MP3（最广泛支持）
- WAV（无损音频）
- OGG（开放音频格式）
- FLAC（无损音频压缩）
- M4A（AAC音频）

#### 3.5.2 音乐信息读取

导入音乐文件时，系统会解析以下信息：

| 信息 | 来源 | 说明 |
|------|------|------|
| 标题 | 文件名或ID3标签 | 歌曲名称 |
| 艺术家 | ID3标签 | 演唱者 |
| 专辑 | ID3标签 | 所属专辑 |
| 时长 | 文件元数据 | 播放时长 |
| 封面 | ID3标签 | 专辑封面图片 |
| 标签 | 用户设置 | 关联情绪标签 |

#### 3.5.3 封面图片处理

1. 从ID3标签中提取APIC（附件图片）数据
2. 将图片数据转换为Blob URL
3. 在播放器、推荐列表、全屏播放器中展示封面
4. 如果没有封面，显示默认图标
